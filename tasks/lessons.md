## Odra / Casper
- **🔴 Native Windows = DEAD-END.** `odra-test` OdraVM'i çalıştırsa bile transitif olarak `casper-execution-engine` → `casper-types 6.1.0`'ı DERLER, o da Unix-only syscall kullanır (`libc::sysconf`/`_SC_PAGESIZE`, `OpenOptions::mode(0o600)`) → Windows MSVC'de compile FAIL. Yani hem test hem `cargo odra build` Windows'ta çalışmaz. **Casper geliştirme Linux/WSL2'de yapılır.** (DÜZELTME: "OdraVM testi wasm gerektirmez" doğru ama Windows'u kurtarmıyor — casper engine yine derleniyor.)
- **WSL2 kurulum** (Ubuntu 24.04): `apt install build-essential pkg-config libssl-dev cmake wabt binaryen git curl` (root: `wsl -u root`, şifresiz); `rustup --default-toolchain nightly-2026-01-01 --profile minimal`; `rustup target add wasm32-unknown-unknown`; `cargo install cargo-odra --locked`. **Kaynak /mnt/c'de** (Windows Write/Edit araçları çalışsın), **build WSL'de** `CARGO_TARGET_DIR` ext4'e yönlendirilerek (hız).
- **CEP-78 modülü yok** (odra-modules 2.8.x). NFT = `Erc721Token` (ERC-721). CEP-78'in özel modları custom build gerektirir.
- **`rust-toolchain` nightly pinli** (`nightly-2026-01-01`); contracts/ içinde cargo o nightly'yi kullanır.
- **cargo-odra native binary** → network (template clone) çalışır; MSYS git DNS sorunu onu etkilemez.
- **🔴 rustc nightly-2026-01-01 borrowck ICE:** HostRef `with_tokens(amount)` çağrısı bir `#[test]` gövdesinde inline kullanılınca (özellikle payable `try_*` entry point + sonradan `registry`'yi tekrar borrow eden testlerde) `mir_borrowck` çöküyor (Vec index panic, `borrowck_check_region_constraints`). Kod hatası DEĞİL, derleyici bug'ı. **Workaround:** her `with_tokens` çağrısını küçük bir helper fn'e izole et (handle'ı orada kur+çağır, sonucu döndür) — with_tokens handle'ını başka şeyler de yapan test gövdesinde tutma. `register_with`/`try_register_with` helper pattern'i identity.rs testlerinde.
- **U256/U512 → `odra::casper_types::{U256, U512}`** (prelude'da YOK). `OdraResult` → `odra::prelude::OdraResult`. `unwrap_or_revert_with(self, Err)` (self = Revertible). `with_tokens` → `odra::host::HostRef` trait import şart.
- **Native CSPR:** payable entry point = `#[odra(payable)]`, `self.env().attached_value() -> U512`, `transfer_tokens(to, amount)`, `self_balance()`. Test: `env.balance_of(addr)`, default hesap bakiyesi = 10e18 motes (devasa). odra_type enum/struct zaten `PartialEq/Eq/Debug` türetir — manuel derive ÇAKIŞIR.
- **🔴 `Erc721Token::mint` owner-gated** (`self.ownable.assert_owner(caller)`); ownable owner = init eden (deployer). SubModule iç çağrıda `self.env().caller()` = dış kullanıcı (IdentityRegistry değil) → **permissionless `register`'da mint FAIL eder.** Ayrıca Erc721Token token_id = U256 (canonical 8004 = u32). **Karar:** permissionless NFT registry için Erc721Token wrap ETME, custom owner-map yaz (u32 native, canonical 8004 method yüzeyi, transferable). odra-modules yine de Escrow'un CEP-18'i için lazım.
- **Events:** `#[odra::event] pub struct E { pub f: T }` (Address Copy'dir, event'te tekrar kullanılabilir), `#[odra::module(events = [E1, E2], errors = Err)]`, `self.env().emit_event(E { .. })`. Test: `env.emitted_event(&ref, E { .. })` (value), `env.emitted(&ref, "E")`, `env.events_count(&ref)`.
- **Cross-contract çağrı:** `use odra::ContractRef;` (prelude'da YOK) → `Cep18ContractRef::new(self.env(), token_addr).transfer_from(&from, &to, &amt)`. Çağıran contract'ın adresi = `self.env().self_address()`. Cross-call'da çağrılan contract'ın `caller()`'ı = çağıran contract (escrow), yani CEP-18 `transfer_from` allowance[client][escrow] kontrol eder → client önce escrow'a `approve` etmeli.
- **`.address()` HostRef'te** `Addressable` trait'inden gelir → test'te `use odra::prelude::Addressable;` şart. Test'te `Address` tipi de `odra::prelude::Address` (contract modülü `prelude::*` alır ama `#[cfg(test)] mod tests` ayrı scope).
- **CEP-18 escrow pattern:** `Cep18::deploy(env, Cep18InitArgs{symbol,name,decimals,initial_supply})` → initial_supply **caller'a** mint edilir (token'ı client olarak deploy et). PRISM checks-effects-interactions: job'u kaydet → SONRA `transfer_from` (transfer revert ederse tüm tx atomik geri döner).
- **🔴 `cargo odra build` + CARGO_TARGET_DIR ÇAKIŞIYOR:** cargo-odra wasm'ı proje `target/`'ında arar; CARGO_TARGET_DIR ext4'e yönlendirilince bulamaz (`cp: cannot stat`). **Build için CARGO_TARGET_DIR'ı UNSET et** (test için kalsın). Build /mnt/c'de yavaş ama tek seferlik.

## Off-chain SDK (TypeScript / casper-js-sdk@5)
- **🔴 casper-js-sdk getDictionaryItemByIdentifier "not found" error shape:** an absent dictionary item throws a PLAIN `Error` (NOT a typed RpcError). `e.code` is **undefined**; the JSON-RPC code -32003 (QueryFailed) lives at **`e.statusCode`** AND **`e.sourceErr.code`**, and is embedded in `e.message` ("Code: -32003, err: Query failed"). To detect absent keys: `const code = e?.code ?? e?.statusCode ?? e?.sourceErr?.code; if (code === -32003) return null;`. Checking only `e.code` silently re-throws on every absent key.
- **🔴 Mock must mirror the REAL error shape.** A not-found unit test mocked `{ code: -32003 }`, which passed but did NOT match the runtime object (`{statusCode, sourceErr:{code}}`) — false green. When a fix changes not-found/error-handling behaviour, the mock must replicate the true thrown object, and you MUST run the LIVE test, not just the mock. (A code-only fix shipped a regression because live was skipped after the change.)
- **Odra reads are wallet-free storage decodes**, verified live: hash-<pkg> -> contract -> "state" URef -> dict item -> strip 4B List<U8> prefix -> bytesrepr. Field indices have a uniform +1 offset vs Rust source declaration order (Odra writes admin at live idx 0).
